## Game tab — Projection model

### Pace × ORTG decomposition
```
league_ortg = 107.3   # 2026 league avg, computed from pace_stats.csv
pace        = (home_pace_regressed + away_pace_regressed) / 2   # K=5 prior to league mean

home_off_rtg = league_ortg + h_oRAPM            # lineup's expected offensive efficiency
home_def_rtg = league_ortg − h_dRAPM            # lineup's expected DRTG (opp pts/100)
home_adj_ortg = home_off_rtg + (away_def_rtg − league_ortg)
away_adj_ortg = away_off_rtg + (home_def_rtg − league_ortg)

home_pts = home_adj_ortg × pace / 100  +  HCA/2
away_pts = away_adj_ortg × pace / 100  −  HCA/2
spread   = home_pts − away_pts        # positive = home favored
total    = home_pts + away_pts
win_prob = norm.cdf(spread / SIGMA_MARGIN)
```

### Calibrated constants (2026)
- `PACE_DEFAULT = 80` (per 40 min, counted possessions)
- `LEAGUE_ORTG = 107.3` (live from pace_stats.csv)
- `BASE_PTS = 85.8` (= LEAGUE_ORTG × PACE_DEFAULT / 100)
- `HCA = 2.2` (split evenly across home/away pts in display)
- `SIGMA_MARGIN = 12.5` (margin std around projected spread; was 10.5, too steep)

## PBP possession walker (`_game_presence_from_pbp`)

ESPN-accurate within ±0.01 min across 36 cross-referenced player-games. Key correctness logic:

1. **Sort PBP** by `(period, -clock_sec, action_priority, order_number)`
   - Handles late events appended out of period order (NBA Stats quirk)
   - At same clock: actions (foul, shot) → subs → period markers (causal order)
2. **Interval accounting** for exact minutes (no bucket rounding):
   - Open interval on sub-in / start of period / event-heuristic
   - Close on sub-out / period end / game end
   - `minutes[pid] = Σ (start_clock − end_clock)`
3. **Case A** — SUB OUT for player not in lineup → data lost their sub-in. Open retro interval at period start.
4. **Case B** — SUB IN for player already in lineup → data lost their sub-out. Discard inflated interval.
5. **Event-heuristic** — any non-sub event for a team player → ensures they're tracked as on-floor.
6. **Ghost-lineup post-process** — for any (player, period) with 0 events, discard their period minutes. Catches missing period-boundary sub-outs where lineup gets carried with a player not actually playing.

Walker output: `(presence: {pid: [bool]*40}, minutes: {pid: float})`.

## Pace = counted, not estimated

- **Pace** (in app/pace_stats/game_log) = possessions normalized to 40 min
- **Poss** = raw counted possessions (PBP walker)
- For OT games: `pace = poss × 40 / game_minutes`
- Examples: 1-OT game with 92.5 poss → pace 82.2; regulation game with 80 poss → pace 80.0

`matchup.predict_game_pace_v2()` reads pace_stats.csv directly (Bayesian K=5 shrinkage to league mean).
